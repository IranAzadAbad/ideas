# Contributing Ideas

Thank you for contributing! This guide will help you submit a clear, actionable idea.

## Before You Start

- **Read the goal**: We're looking for ideas that can become real projects, teams, or companies
- **Check discussions**: Browse [GitHub Discussions](https://github.com/IranAzadAbad/ideas/discussions) to see if your idea already exists or is being discussed
- **Review examples**: Look at the `/examples` folder to see what good ideas look like

## How to Submit

### Option 1: Create a Pull Request (Recommended for new ideas)

1. Fork the repository
2. Create a new file in `/ideas` with your idea name: `ideas/your-idea-title.md`
3. Use the [template](templates/idea-template.md)
4. Submit a pull request with a clear description

### Option 2: Start a Discussion

If you want feedback before writing, start a [Discussion](https://github.com/IranAzadAbad/ideas/discussions) first.

## Structure Your Idea

Follow the sections below. Don't worry about perfect prose—clarity matters more than polish.

### Title
- Short (5-10 words), descriptive
- Examples: "Network for diaspora tech talent" not "Connecting people"

### Author (optional)
- Your name, brief background (2-3 sentences), how to contact you

### Abstract (1-2 paragraphs)
- **What is it?** Clear one-sentence summary first
- **Why it matters**: The problem it solves
- **Who benefits**: Who are the primary users/stakeholders?

### Why It's Interesting (1-2 paragraphs)
- **Why now**: What's changed? Why is this timely?
- **Why Iran specifically**: How is this relevant to Iran's context?
- **Expected impact**: What's the realistic upside? (users reached, revenue potential, social impact)

### Approach (Key sections)
- **Key steps**: 3-5 main phases or milestones
- **Timeline**: Realistic timeframe (months/years)
- **Methods/Technology**: How will you build it? What tools?
- **Go-to-market** (if applicable): How will people find and use it?

### Help Needed
- **Be specific**, not vague. Instead of "engineers", write "2-3 backend engineers with Node.js experience"
- **Skill/role categories**:
  - Technical (what type, level)
  - Business/strategy
  - Design
  - Domain expertise
  - Funding
  - Partnerships
  - Data/research
  - Other

### Supporting Material (optional but recommended)
- Diagrams, mockups, sketches
- Links to related projects, research, competitors
- Charts or data illustrating the problem
- Videos or presentations

## Tips for Strong Ideas

✅ **DO:**
- Be concise (1-3 pages is ideal)
- Focus on solving real problems
- Show you understand the context (Iran-specific challenges, regulations, market)
- Break down complex ideas into actionable steps
- Be specific about what help you need
- Use visuals when they clarify (even rough sketches help)

❌ **DON'T:**
- Write vague, aspirational essays ("improve Iran's tech ecosystem")
- Focus on politics rather than solutions
- Propose ideas without understanding the problem
- Ask for help without being specific ("need smart people")
- Ignore feasibility (be realistic about timeline and resources)
- Make claims without context ("will create 10,000 jobs" — how? why?)

---

## Common Pitfalls

| Problem | Fix |
|---------|-----|
| Too vague | Replace "improve X" with "reduce Y by doing Z" |
| No specific help needed | Name roles: "1 iOS dev", "design partner", not "team" |
| Missing Iran context | Add: regulations, market conditions, diaspora dynamics, local needs |
| Timeline is fantasy | Be honest about phases and dependencies |
| No validation | Reference research, similar projects, or user conversations |

## Questions to Ask Yourself

Before submitting, answer these:

1. Can someone unfamiliar with the space understand this in 10 minutes?
2. Would I personally want to work on this?
3. Have I identified the real problem, not a symptom?
4. Are my timelines realistic given the resources needed?
5. Is there evidence this matters to real people in Iran?

## After You Submit
